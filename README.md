# 田川审计视角 Skill

一个给审计实务场景用的中文思维技能包。

它不是教你背准则，也不是帮你把底稿模板再抄一遍，而是把 39 篇“审计笔记”里反复出现的作业逻辑提炼成一套可运行的判断框架：先看业务，再看风险，再看程序，再看证据链，最后看底稿说明有没有把话说圆。

适合这样的场景：

- 复核底稿，判断哪里“看起来做了，实际上没扣题”
- 设计审计程序，避免只会从模板倒推工作量
- 解释异常波动、账龄、毛利、回款、存货、研发费用等问题
- 重写审计说明、结论、问询回复，让逻辑更像审计而不是填表
- 用大白话解释审计风险、认定和证据链

## 这个 Skill 能做什么

这个 skill 的核心不是提供标准答案，而是提供一套审计判断顺序：

1. 先分型：这是底稿复核、程序设计、异常分析，还是问询回复。
2. 再看闭环：业务闭环、会计闭环、风险闭环、证据闭环、底稿闭环。
3. 再检查横向逻辑：前面识别的风险，后面有没有程序接住。
4. 再检查纵向逻辑：目标、过程、说明、结论是不是同一件事。
5. 最后输出改法：补什么事实、加什么程序、说明该怎么重写。

说白了，它擅长的不是“把话说漂亮”，而是把“这张底稿到底要证明什么”这件事掰清楚。

## 目录结构

这个 skill 的最小可运行结构如下：

```text
tianchuan-audit-perspective/
├─ SKILL.md
└─ references/
   ├─ research/
   └─ sources/
```

其中：

- SKILL.md：技能入口文件，宿主通过它识别 skill 的名称、描述、工作流和边界
- references/research/：提炼后的语料分析、心智模型、表达 DNA 和工作流
- references/sources/articles/：39 篇原始审计笔记 Markdown 资料

## 安装方法

这是一个本地自定义 skill。安装时不要写死某个具体客户端名称，按你当前宿主终端的技能目录约定放置即可。

常见有两种放法：

1. 放到宿主的全局 skills 目录，作为全局可用 skill
2. 放到当前项目的 skills 目录，作为项目内可用 skill

具体目录名以你的宿主终端要求为准，但有一个通用原则：

- 整个 tianchuan-audit-perspective 文件夹要作为一个完整 skill 目录存在
- 目录根部必须直接能看到 SKILL.md
- 不要把 SKILL.md 再多包一层

### 方式一：放到宿主的全局目录

1. 找到宿主终端要求的全局 skills 目录。

2. 把整个文件夹放进去，形成类似这样的结构：

```text
[global-skills-dir]/tianchuan-audit-perspective/SKILL.md
```

3. 确认目录根部能直接看到 SKILL.md，不要多套一层文件夹。

正确结构：

```text
[skills-dir]/tianchuan-audit-perspective/SKILL.md
```

错误结构：

```text
[skills-dir]/tianchuan-audit-perspective/tianchuan-audit-perspective/SKILL.md
```

4. 保留 references 目录，别只复制 SKILL.md。虽然 skill 主入口是 SKILL.md，但研究和来源资料最好一起带上，后续维护也更方便。

5. 重新打开一个新的对话，或按宿主终端的要求刷新技能扫描。

### 方式二：放到项目目录

1. 在当前项目下找到宿主终端支持的项目级 skills 目录。

2. 把整个文件夹放进去，形成类似这样的结构：

```text
[project-root]/[project-skills-dir]/tianchuan-audit-perspective/SKILL.md
```

3. 确认 SKILL.md 在 skill 根目录第一层，references 目录也一并保留。

4. 重新打开当前项目会话，或按宿主终端的要求刷新技能加载。

### 方式三：从压缩包安装

1. 解压压缩包。
2. 按上面两种方式之一，把解压后的整个文件夹放到正确的全局目录或项目目录。
3. 解压后检查一遍根目录层级，确保 SKILL.md 在第一层。
4. 新开对话验证是否已加载。

## 怎么验证安装成功

安装完成后，可以直接在对话里提这类问题：

- 按田川的视角，帮我看这张收入底稿最大的问题是什么
- 这个存货监盘底稿为什么看起来做了很多，实际上逻辑没闭环
- 应收账款函证回函差异，底稿说明应该怎么写
- 销售费用波动很大，先做什么分析程序，后面接什么实质程序
- 别按模板，按风险和证据链重写这段审计说明

如果 skill 被正确识别，输出会明显带有这几个特征：

- 先说问题，不绕
- 强调横向逻辑和纵向逻辑
- 会追问“这张底稿到底要证明什么”
- 不鼓励只抄模板或只写“未见异常”
- 更偏业务闭环和证据链，而不是空泛套话

## 使用建议

为了让这个 skill 发挥得更像样，提问时最好给一点现场信息，而不是只丢一句“这个科目怎么审”。

更好的问法包括：

- 科目和期间：比如收入、存货、研发费用、现金流附表
- 当前异常：比如毛利率下降、账龄拉长、回款异常、盘亏盘盈
- 已做程序：比如已经抽凭、已经函证、已经做分析程序
- 想要产出：比如重写底稿说明、设计进一步程序、复核执行逻辑

这个 skill 会更像一个“审计经理视角的逻辑复核器”，而不是百科问答机。

## 适用边界

这个 skill 适合做方法判断，不适合直接替代这些东西：

- 最新准则全文和逐条适用
- 具体项目上的事务所质控结论
- 最新监管案例、处罚口径、行业特殊规则
- 缺少事实时直接替你编造底稿说明

如果问题依赖具体公司、行业、政策或最新监管动态，正确做法不是硬答，而是先补事实，再用这套框架判断。

## 常见问题

### 1. 放进目录了，但没触发

先检查三件事：

1. 目录名是否完整
2. 根目录下是否直接存在 SKILL.md
3. 是否开启了新对话或重启了宿主

### 2. 为什么不建议只复制 SKILL.md

因为这个 skill 不是一句提示词，它背后还有研究材料和来源语料。只留入口文件虽然可能勉强可用，但后续更新、复核和追溯会很难受。

### 3. 这个 skill 能不能替代注册会计师判断

不能。它提供的是一种高密度、偏实务的思考路径，不是职业责任替代品。
