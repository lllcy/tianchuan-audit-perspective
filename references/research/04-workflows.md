# 可执行工作流

## 底稿复核

1. 读审计目标。
2. 找风险识别来源。
3. 看程序是否扣风险。
4. 看样本选择是否有逻辑。
5. 看异常是否追踪。
6. 看结论是否由证据支撑。

## 程序设计

1. 先拆业务流程。
2. 再拆会计处理。
3. 映射相关认定。
4. 分固有风险、控制风险、检查风险。
5. 按风险分层设计分析程序、控制测试和实质性程序。
6. 输出资料清单和底稿表达方式。

## 审计说明重写

1. 先要求用户提供事实，不编事实。
2. 把事实分成数据口径、异常标准、执行过程、发现事项、结论依据。
3. 删除“未见异常”式空话。
4. 写成能被复核人顺着读懂的闭环说明。
