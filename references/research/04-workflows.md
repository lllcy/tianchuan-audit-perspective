# 可执行工作流

## 底稿复核

1. 读审计目标。
2. 找风险识别来源。
3. 看程序是否扣风险。
4. 看样本选择是否有逻辑。
5. 看异常是否追踪。
6. 看结论是否由证据支撑。

### 底稿复核增强流程

1. **先看目标**：这张底稿要证明什么认定、解决什么风险；目标不清先改目标。
2. **再看表格字段**：字段是否服务于目标；无用字段删减，缺失字段补充。
3. **做面上识别**：从金额占比、结构变化、期间变动、主要项目、异常项目识别风险标签。
4. **分一般样本和特殊样本**：一般样本控制检查风险，特殊样本回应特殊风险或舞弊风险。
5. **检查程序递推**：明细表发现的问题，后续细节测试、访谈、函证、合同、外部信息、期后检查是否接住。
6. **看证据链强度**：访谈、内部单据、外部证据、资金流、物流/交付、系统记录是否互相印证。
7. **看说明是否收口**：异常标准、执行过程、发现事项、结论依据是否写清楚；不能只写“未见异常”。
8. **给修改意见**：按“改目标/改口径/补索引/补程序/删空表/重写说明/扩围或裁剪样本”输出。

## 程序设计

1. 先拆业务流程。
2. 再拆会计处理。
3. 映射相关认定。
4. 分固有风险、控制风险、检查风险。
5. 按风险分层设计分析程序、控制测试和实质性程序。
6. 输出资料清单和底稿表达方式。

## 审计说明重写

1. 先要求用户提供事实，不编事实。
2. 把事实分成数据口径、异常标准、执行过程、发现事项、结论依据。
3. 删除“未见异常”式空话。
4. 写成能被复核人顺着读懂的闭环说明。

## 分科目底稿复核提示

- **收入**：合同检查要看退货、返利、暂定价格、可变对价、履约义务和确认时点；临近期末大额收入要补过程痕迹和特殊应对。
- **固定资产**：明细表要兼顾附注披露和发生额调整；关注转固时点、折旧年限一致性、产能利用率、募投/在建合理性。
- **货币资金**：账户清单、对账单、函证数量和余额必须一致；余额变动要结合现金流和借款，不用零星收付款解释整体变化。
- **研发费用**：项目、人员、工时、费用口径、监管线和差异原因要说清；触及研发投入比例或金额红线时，错报容忍度下降。
- **销售费用**：合并层面先看主要项目；会务、推广、样品等真实性要组合外部公开信息、供应商信息、差旅和领用记录。
- **存货**：先看产销存和结存变化，再看库龄、周转、计价和委外加工；金额重大且周转异常的项目优先。
