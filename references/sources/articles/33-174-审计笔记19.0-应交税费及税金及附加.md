# 审计笔记19.0-应交税费及税金及附加

首先，很负责任的说，税是我的短板...以下分享仅限于审计作业逻辑，不涉及税收征管或计税逻辑...



（用电脑看...用电脑看...手机看眼睛会瞎的...)

应交税费和税金及附加科目本身是由税法规范的，下面可能不会过多的展开去写税法的相关规定...主要还是聚焦执行税金相关科目审计过程中，可能出现的幺蛾子...

一、增值税

（一）核对纳税申报表

虽然底稿编制要求中需要作业人员对应税项目进行核对，但是个人建议上是先看眼纳税申报表的余额...因为余额会涉及到科目的列报，很多时候应交税费的余额到底是借方余额还是贷方余额并不以账面为准的...

why？因为这几年经济形式不好，很多企业在不能保证足额收款前，会选择暂估收入的方式，但是财务老师在进行账务处理时选择的应交税费-应交增值税-销项税科目，而未使用代转销项税，就导致账载销项税大于实际纳税申报，就可能会出现账载为贷方结余，但申报表是留抵税额或应缴税额小于账载金额的囧象...反过来也一样，不过概率相对前者要小很多，就是暂估进项税...这种情况一般发生的概率极低，但并不是没有，所以如果发现账载增值税为借方余额，但纳税申报表是应缴税额或留抵税额小于账载余额时，可以针对性的检查一下....

（二）纳税申报收入检查

纳税申报表表检查还有一项是纳税申报收入的检查，非常建议按月登记，再逐月检查，因为如果拿全年纳税申报收入去核对的话，大概率会对不上...到头来还是得按月拆开去做进一步核对...

为什么会对不上？一方面就是像前面提到的，收入确认和纳税义务有时候并不是同步的，特别是以时段法确认收入的行业，为避免缴税带来的现金流压力，有时会选择暂估收入而不进行纳税申报，合规性的问题不做进一步讨论...来说一说如何发现可能存在的舞弊或重大错报风险...

1.建议结合以前年度底稿整体识别风险

为什么非要把以前年度拉上，是因为单独一个会计期间往往会导致核查的“视野”受到局限，讲人话就是12个月的趋势可能不足以识别到风险...具体一点来说，就是被审计单位是否长期存在会计收入大于税务收入情形？为什么要特别强调长期，是因为会计口径收入和税务口径收入往往是一个时间性差异，在暂估收入期间，会计收入大于纳税申报收入，在开票结算并完成纳税申报期间，会计收入小于纳税申报收入，所以客观上讲，除非企业的业务呈一个明显、持续的上升趋势，在业务收入规模稳定的情况下，会计收入与纳税申报收入应该是此消彼长的，不会出现长期会计收入大于纳税申报收入情形。

如果出现了长期存在会计收入大于税务收入情形那？好一点的是企业提前确认收入，实际工作量或确认收入节点未完成相应履约义务，坏一点的话，可能收入是假的、客户也是假的....比较洗收入规模的税务成本还是很高的，特别是对于高毛利行业...自然而然的就需要扩大对收入的核查范围和核查力度...

如果反过来，长期存在会计收入小于税务收入情形那会不会好一些？只能说相对好，比较是低估收入，但是也不排除因为行业特性和客户群体导致的特殊性，因为很多大型集团、政府执行预算制度，可能会要求供应商提前开票申请预算，就导致纳税收入时点提前。如果会计上按照准则相关要求未进行会计处理，实属应该给财务老师磕一个...要不光调这些提前开票的收入确认就够吃一壶的...

2.出口报关收入核查

对于IPO项目，特别是外销收入占比较高的发行人，在反馈问询过程中可能会要求中介机构补充出口报关收入（税务口径）与外销收入（会计口径）的核查程序，这个程序本质上也不复杂，就是从纳税申报系统中导出出口报关收入明细，再和收入大表做核对就好...

考虑到核查需要，如果涉及重大比例的外销，在执行实质性程序或准备收入成本大表过程中，最好提前把报关信息的唯一标识提前备出来，要不后面反馈的时候现搂，还是有点压力的...特别是翻出一些奇奇怪怪的东西...

因为外销收入，特别是DAP等特殊模式，可能会导致出口报关收入与外销收入不一致情形，但是这种不是最可怕的...可怕的是会计确认了收入，但是木有报关收入...反过来会好一些？其实也不是很好，就比如公司想隐匿一些非合并关联交易，就可能会通过不进行会计出来的方式，但是在核对报关收入的时候就会冒出来...

增值税总得来说最怕的还是会计收入长期或连续多期大于纳税申报收入，毕竟这可能会涉及到收入的发生、截止认定，甚至是可能存在的真实性问题...同时，企业也可能会涉及税务风险...所以，如果碰到了，神经还是需要绷紧一点的...

除了上面发生额的问题，如果企业长期存在大额的“待认证”进项税（其他流动资产，未进行税务认证的“进项税”），注意，不是待抵扣，是待认证，也需要考虑该部分“进项税”所对应的成本、费用是否真实...why？那个正经供应商不希望早点开票早点收钱...如果开了票但是财务不做认证那？那只能说财务老师的思路有点清奇...当然，也在一定程度上反应相关的“进项税”可能是虚的，至于相关成本、费用的真实性是否真的有问题，还是需要通过执行进一步审计程序才能核实，单纯以上述情形判断为舞弊...约等于是算命....

另外，说个题外话，就是审计调整是否要调税的问题...个人一直觉得审计调整调增值税是一件很蛋疼的事，很多时候成本、费用可以暂估，不调整进项税...但为啥偏偏到收入这里就要补销项税...当然，每个项目组有每个项目组的考虑，这里就不作延申讨论了...
